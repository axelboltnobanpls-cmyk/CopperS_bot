# 🤖 Бот для бесплатных ключей — CopperShop    by. Arferno

Полностью рабочий Telegram-бот, который выдаёт бесплатные ключи за подписку на канал.  
Акция одноразовая — один ключ на человека.

---

## 📁 Структура проекта

```
CopperS_bot/
├── bot.py              — основной файл бота
├── maintenance.py      — режим технического обслуживания
├── keys.json           — база ключей (загружаете сюда свои)
├── users.db            — база данных пользователей (создаётся автоматически)
├── config.json         — конфигурация (сохраняет канал)
├── requirements.txt    — зависимости Python
└── README.md           — эта инструкция
```

## 🧰 Шаг 1: Подготовка

### 1.1 Установите Python
Скачайте и установите Python 3.10+ с официального сайта:  
👉 https://www.python.org/downloads/  
При установке **обязательно** поставьте галочку **«Add Python to PATH»**.

### 1.2 Создайте бота у @BotFather
1. Откройте Telegram и найдите **@BotFather**
2. Отправьте `/newbot`
3. Задайте имя бота (например: `Copper Keys Bot`)
4. Задайте юзернейм (например: `coppers_keys_bot`)
5. Скопируйте полученный **токен** — он выглядит примерно так:  
   `7123456789:AAF123AbCdEfGhIjKlMnOpQrStUvWxYz`

### 1.3 Сделайте бота администратором канала
1. Откройте канал **@copperS_shop** (или создайте новый)
2. Зайдите в **Настройки канала → Администраторы**
3. Добавьте вашего бота и дайте ему права **администратора**  
   (минимум: права на чтение сообщений и список участников)  
   ⚠️ **Без прав администратора проверка подписки НЕ будет работать!**

## 🛠️ Шаг 2: Настройка кода

### 2.1 Откройте файл `bot.py` в любом текстовом редакторе
Рекомендуется: **VS Code** (бесплатный) или **Notepad++**

### 2.2 Измените настройки (в начале файла)


**Как узнать свой Telegram ID?**  
Напишите боту **@userinfobot** в Telegram — он покажет ваш ID.

## 📦 Шаг 4: Загрузка ключей

### Через Telegram (команда админа):
```
/addkey
ABCD-1234-EFGH
IJKL-5678-MNOP
```
Бот добавит ключи в базу и покажет сколько теперь доступно.

### Через файл `keys.json`:
```json
{
  "keys": [
    "XXXX-XXXX-XXXX",
    "YYYY-YYYY-YYYY"
  ]
}
```

### Проверить остаток:
```
/stats
```

### Как работает система ключей:
- Ключи хранятся в файле `keys.json` (порядок — очередь FIFO)
- При выдаче ключ **удаляется** из файла → никто больше не получит его
- Все выданные ключи записываются в `users.db` с привязкой к пользователю
- Даже при перезапуске бота ключи не дублируются — используется SQLite

## 👤 Как это работает для пользователя

### Сценарий 1: Пользователь НЕ подписан
1. Пользователь нажимает `/start`
2. Бот пишет: «Для получения ключа подпишитесь на канал» + показывает кнопку
3. Пользователь нажимает кнопку → переходит в канал → подписывается
4. Возвращается в бота → нажимает «✅ Проверить подписку»
5. Бот проверяет подписку → если всё ОК → выдаёт ключ

### Сценарий 2: Пользователь уже подписан
1. Пользователь нажимает `/start`
2. Бот мгновенно проверяет подписку → видит что подписан → **сразу выдаёт ключ**
3. Пользователь получает ключ + инструкцию

### Сценарий 3: Ключи закончились
1. Пользователь нажимает `/start`
2. Бот пишет: «Ключи временно закончились, пожалуйста зайдите позже. Чтобы следить за новостями — подпишитесь: t.me/copperS_shop»

## 🔧 Режим технического обслуживания (`maintenance.py`)

Когда бот на обновлении и ключи нельзя выдавать — используйте `maintenance.py`.

### Что делает:
- ❌ Ключи **НЕ выдаются**
- ✅ Все ключи в `keys.json` **сохраняются** (не удаляются!)
- ✅ Пользователи видят: *«Технический перерыв, делаем обновления»*
- ✅ Админ может проверить `/stats` и `/listkeys`

### Запуск:
```bash
python maintenance.py
```

### Остановка:
- Нажмите `Ctrl + C` в терминале
- Запустите `python bot.py` обратно — ключи будут на месте!

2. Нажмите **«New Project»**
3. Выберите **«Deploy from GitHub Repo»**
4. Найдите и выберите репозиторий `copper-keys-bot`

1. В Railway → **Settings → Variables**

```python
BOT_TOKEN = os.environ.get("BOT_TOKEN", "СТАРЫЙ_ТОКЕН_НА_СЛУЧАЙ")
```
(`import os` вверху уже есть ✅)

## 🆘 Частые проблемы

**В:** Бот не запускается на Railway
**О:** Проверьте, что переменная `BOT_TOKEN` добавлена в Settings → Variables.

**В:** Бот выдаёт ошибку «conflict: terminated by other getUpdates»
**О:** Бот запущен в двух экземплярах. Нажмите **Redeploy** на Railway.

**В:** Ключи пропали после перезапуска
**О:** Загружайте ключи через `/addkey`.

**В:** Бот не проверяет подписку
**О:** Бот должен быть **администратором** в канале.
