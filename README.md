# 🤖 Бот для бесплатных ключей — CopperShop    by. Arferno

Полностью рабочий Telegram-бот, который выдаёт бесплатные ключи за подписку на канал.  
Акция одноразовая — один ключ на человека.

---

## 📁 Структура проекта

```
CopperS_bot/
├── bot.py              — основной файл бота (выдача ключей, проверка подписки)
├── maintenance.py      — режим технического обслуживания
├── admin_panel.py      — админ-панель (отдельный скрипт)
├── keys.json           — база ключей (загружаете сюда свои)
├── users.db            — база данных пользователей (создаётся автоматически)
├── config.json         — конфигурация (сохраняет канал)
├── run_all.py          — запуск основного бота + админ-панели одним сервисом
├── requirements.txt    — зависимости Python
└── README.md           — эта инструкция
```

## 🧰 Шаг 1: Подготовка

### 1.1 Установите Python
Скачайте и установите Python 3.10+ с официального сайта:  
👉 https://www.python.org/downloads/  
При установке **обязательно** поставьте галочку **«Add Python to PATH»**.

### 1.2 Создайте ботов у @BotFather
1. Откройте Telegram и найдите **@BotFather**
2. Отправьте `/newbot`
3. Задайте имя бота (например: `Copper Keys Bot`)
4. Задайте юзернейм (например: `coppers_keys_bot`)
5. Скопируйте полученный **токен** — он выглядит примерно так:  
   `7123456789:AAF123AbCdEfGhIjKlMnOpQrStUvWxYz`

> ⚠️ **Нужно создать двух ботов**: основного и для админ-панели (у каждого свой токен).  
> Один токен нельзя использовать в двух ботах одновременно.

### 1.3 Сделайте бота администратором канала
1. Откройте канал **@copperS_shop** (или создайте новый)
2. Зайдите в **Настройки канала → Администраторы**
3. Добавьте вашего бота и дайте ему права **администратора**  
   (минимум: права на чтение сообщений и список участников)  
   ⚠️ **Без прав администратора проверка подписки НЕ будет работать!**

### 1.4 Узнайте свой Telegram ID
Напишите боту **@userinfobot** в Telegram — он покажет ваш ID.

## 🚀 Шаг 3: Запуск на локальном компьютере

### 3.1 Откройте терминал (командную строку)

**Windows:**  
Нажмите `Win + R`, введите `cmd`, нажмите Enter.  
Перейдите в папку бота:
```
cd D:\CopperS_bot
```

### 3.2 Установите зависимости
```bash
pip install -r requirements.txt
```

### 3.3 Запуск основного бота
```bash
python bot.py
```
Вы увидите в консоли:
```
🚀 Бот запущен!
📢 Канал для проверки: @copperS_shop
🔑 Ключей в базе: 0
```

### 3.4 Запуск админ-панели (в отдельном терминале!)
```bash
python admin_panel.py
```
> Откройте **второе окно** терминала — иначе бот «займёт» консоль и второй не запустится.

### 3.5 Режим технического обслуживания

Когда нужно остановить выдачу ключей, но сохранить их:

1. **Остановите основного бота** — нажмите `Ctrl + C` в его терминале
2. **Запустите maintenance:**
```bash
python maintenance.py
```
Теперь пользователи будут видеть: *«Технический перерыв, делаем обновления»*, а ключи в `keys.json` **не пропадут**.

3. **Чтобы вернуть бота обратно:**
   - Остановите maintenance (`Ctrl + C`)
   - Запустите `python bot.py` — ключи будут на месте!

## 📦 Шаг 4: Загрузка ключей

### 4.1 Через файл `keys.json`
Откройте файл и запишите ключи, каждый в кавычках, через запятую:

```json
{
  "keys": [
    "ABCD-1234-EFGH-5678",
    "IJKL-9012-MNOP-3456",
    "QRST-7890-UVWX-1234"
  ]
}
```

### 4.2 Через Telegram (команда админа)
Отправьте **админ-боту** в личку:
```
/addkey
XXXXX-XXXXX-XXXXX
YYYYY-YYYYY-YYYYY
```
Бот добавит ключи и покажет, сколько теперь доступно.

### Как работает система ключей:
- Ключи хранятся в файле `keys.json` (очередь FIFO)
- При выдаче ключ **удаляется** из файла → никто больше не получит его
- Все выданные ключи записываются в `users.db` с привязкой к пользователю
- Даже при перезапуске бота ключи не дублируются

## 👤 Как это работает для пользователя

### Сценарий 1: Пользователь НЕ подписан
1. Пользователь нажимает `/start`
2. Бот пишет: «Для получения ключа подпишитесь на канал» + показывает кнопку
3. Пользователь нажимает кнопку → переходит в канал → подписывается
4. Возвращается в бота → нажимает «✅ Проверить подписку»
5. Бот проверяет подписку → если всё ОК → выдаёт ключ

### Сценарий 2: Пользователь уже подписан
1. Пользователь нажимает `/start`
2. Бот мгновенно проверяет подписку → видит что подписан → **сразу выдаёт ключ**

### Сценарий 3: Ключи закончились
1. Пользователь нажимает `/start`
2. Бот пишет: «Ключи временно закончились, пожалуйста зайдите позже. Чтобы следить за новостями — подпишитесь: t.me/copperS_shop»

### Шаг 6: Деплой одним сервисом (основной бот + админ-панель)

Это самый простой вариант — **оба бота запускаются в одном сервисе** через `run_all.py`:

**Start Command:**
```
python run_all.py
```

Готово! Railway запустит и основного бота, и админ-панель одновременно. В Telegram:
- Основной бот — работает с пользователями (`/start`)
- Админ-панель — `/admin` в личку админ-боту

> **Минус:** если один бот упадёт — второй тоже остановится.

---

### Шаг 6А: Деплой двумя отдельными сервисами (продвинутый вариант)

Если хотите, чтобы админ-панель работала **независимо** от основного бота:

1. В Railway → **New Service**
2. Выберите **тот же репозиторий** `copper-keys-bot`
3. В **Start Command** первого сервиса укажите: `python bot.py`
4. В **Start Command** второго сервиса укажите: `python admin_panel.py`
5. В **Settings → Variables** второго сервиса добавьте:

| Имя | Значение |
|-----|----------|
| `ADMIN_TOKEN` | `токен админ-бота от BotFather` |

6. Нажмите **Deploy** на каждом сервисе

Теперь у вас **два сервиса** в Railway:

| Сервис | Команда | Что делает |
|--------|---------|------------|
| `copper-main` | `python bot.py` | Работает с пользователями |
| `copper-admin` | `python admin_panel.py` | Админ-панель для вас |

---

### Управление сервисами

| Действие | Как сделать |
|----------|-------------|
| **Остановить основного бота** | Railway → сервис `copper-main` → **Stop** |
| **Запустить основного бота** | Railway → сервис `copper-main` → **Start** |
| **Перезапустить бота** | Railway → **Redeploy** |

| **Вернуть бота обратно** | Start Command → `python bot.py` → Redeploy |
| **Открыть админ-панель** | Напишите `/admin` админ-боту в Telegram |
| **Обновить код** | `git push` → все сервисы обновятся автоматически |

## 🔑 Полезные команды (в личку боту):

| Команда | Описание |

## 🆘 Частые проблемы

**В:** Бот не запускается на Railway  
**О:** Проверьте, что переменная `BOT_TOKEN` добавлена в Settings → Variables.

**В:** Бот выдаёт ошибку «conflict: terminated by other getUpdates»  
**О:** Бот запущен в двух экземплярах. Нажмите **Redeploy** на Railway.

**В:** Ключи пропали после перезапуска  
**О:** Загружайте ключи через `/addkey` — они сохраняются в `keys.json`.

**В:** Бот не проверяет подписку  
**О:** Бот должен быть **администратором** в канале.
